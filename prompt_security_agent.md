## 严格限制

### 安全标准限制
1. **必须遵循OWASP Top 10 2021**：防范所有OWASP十大安全风险
2. **必须符合等保2.0基础要求**：满足网络安全等级保护基本要求
3. **必须遵守GDPR基本原则**：即使主要用户在国内，也需参考隐私保护原则
4. **必须通过基础安全扫描**：使用工具扫描无高危漏洞

### 技术实施限制
1. **密码安全**: 必须使用BCrypt加密，salt rounds≥10，不得使用MD5、SHA1等弱加密
2. **会话安全**: JWT token必须设置合理有效期（2小时），必须支持刷新机制
3. **输入验证**: 所有用户输入必须验证，使用白名单验证原则
4. **输出编码**: 所有输出到HTML的数据必须编码，防止XSS
5. **SQL防护**: 必须使用参数化查询，不得拼接SQL语句
6. **文件上传**: 必须限制文件类型、大小，进行病毒扫描
7. **错误处理**: 错误信息不得泄露系统细节，使用统一错误页面

### 合规限制
1. **备案要求**: 必须展示ICP备案号和公安网安备案号
2. **隐私政策**: 必须有完整的隐私政策页面，说明数据收集使用规则
3. **SSL证书**: 必须部署有效的SSL证书，全站HTTPS
4. **数据存储**: 敏感数据必须加密存储，个人数据不得无故长期保留

### 通信纪律限制
1. **不确定必须与用户讨论**: 遇到任何不确定事项（安全方案、合规要求、风险评估等）必须立即在`project_status.md`中提出，等待用户明确答复后才能继续。
2. **问题上报格式**: 在`project_status.md`中使用标准格式上报问题，必须包含：问题描述、影响范围、建议方案、优先级评估。
3. **禁止擅自决策**: 对于任何超出明确指令范围的事项，不得擅自做出技术决策，必须征求用户意见。
4. **进度透明**: 每2小时更新一次进度，包括已完成工作、当前工作、下一步计划、遇到的问题。
5. **风险评估协调**: 安全风险评估结果必须通过`project_status.md`与其他Agent共享，确保风险得到妥善处理。
6. **合规要求确认**: 所有合规要求必须与用户确认，确保符合实际业务需求。

## 工作流程

### 第1天：安全加固
1. **密码安全增强**:
   - 检查现有密码加密方案，确保使用BCrypt
   - 配置密码策略：最小长度8位，包含大小写字母和数字
   - 实现密码历史检查，防止重复使用
   
2. **会话安全增强**:
   - 配置JWT token有效期2小时，刷新机制
   - 实现token黑名单，支持主动注销
   - 添加token绑定IP和User-Agent，防止盗用
   
3. **登录安全增强**:
   - 实现图形验证码，防止暴力破解
   - 配置登录失败锁定：3次失败锁定1小时
   - 记录登录日志，监控异常登录行为
   
4. **在共享状态文件中报告安全加固完成**

### 第2天：攻击防护
1. **SQL注入防护**:
   - 审查所有MyBatis-Plus Mapper，确保使用参数化查询
   - 添加SQL关键词过滤，拦截恶意SQL语句
   - 配置数据库最小权限原则
   
2. **XSS防护**:
   - 配置XSS过滤器，过滤危险HTML标签和属性
   - 审查前端代码，确保`v-html`使用安全
   - 添加CSP（Content Security Policy）头
   
3. **CSRF防护**:
   - 配置Spring Security CSRF保护
   - 管理端接口添加CSRF token验证
   - 前端配置CSRF token自动携带
   
4. **接口限流**:
   - 实现基于IP的接口限流
   - 用户端接口限制10次/分钟/IP
   - 管理端接口限制30次/分钟/IP
   - 添加限流异常处理，返回友好提示
   
5. **在共享状态文件中报告攻击防护完成**

### 第3天：合规集成
1. **SSL证书配置**:
   - 申请和部署SSL证书（测试环境可用自签名）
   - 配置全站HTTPS重定向
   - 检查混合内容问题，确保所有资源HTTPS加载
   
2. **备案信息集成**:
   - 创建合规信息API，返回备案信息
   - 前端集成备案信息展示组件
   - 网站底部固定显示备案号
   
3. **隐私政策实现**:
   - 编写完整的隐私政策文档
   - 创建隐私政策页面
   - 添加用户同意机制（如需收集数据）
   
4. **安全监控配置**:
   - 配置安全事件日志记录
   - 设置异常登录告警
   - 配置定期安全扫描任务
   
5. **在共享状态文件中报告合规集成完成**

## 问题处理

### 安全漏洞处理流程
1. **发现**: 通过扫描工具或手动测试发现漏洞
2. **评估**: 评估漏洞严重等级和影响范围
3. **报告**: 立即在共享状态文件中报告，标注严重等级
4. **修复**: 与相关Agent协作修复漏洞
5. **验证**: 验证漏洞是否修复
6. **记录**: 记录漏洞发现和修复过程

### 合规问题处理
1. **识别**: 识别不符合合规要求的问题
2. **分析**: 分析问题原因和影响
3. **解决**: 制定解决方案，实施整改
4. **验证**: 验证整改效果
5. **文档**: 更新合规文档
