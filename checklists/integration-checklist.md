# 集成检查清单

## 概述
- **用途**: 确保项目各模块正确集成，接口一致，数据流畅通
- **适用阶段**: 开发阶段、测试阶段、部署阶段
- **负责人**: integration Agent

## 接口集成检查
### 前后端接口检查
- [ ] 接口URL与接口规范一致
- [ ] 请求方法（GET/POST/PUT/DELETE）正确
- [ ] 请求参数名称、类型、是否必需与规范一致
- [ ] 响应格式符合规范（success, data, message, code）
- [ ] 错误处理机制一致（HTTP状态码、错误消息格式）
- [ ] 分页参数（page, size）处理一致
- [ ] 排序参数（sortBy, order）处理一致

### 数据格式检查
- [ ] 日期时间格式统一（ISO 8601格式）
- [ ] 数字格式统一（整数、小数精度）
- [ ] 布尔值格式统一（true/false）
- [ ] 空值处理统一（null vs 空字符串 vs 空数组）
- [ ] 枚举值定义一致

## 数据库集成检查
- [ ] 数据库表结构与schema.sql一致
- [ ] 外键约束正确配置
- [ ] 索引配置合理
- [ ] 数据类型与实体类字段类型匹配
- [ ] 软删除机制一致（deleted_at字段）
- [ ] 数据迁移脚本完整

## 服务集成检查
- [ ] 高德地图API集成正确
- [ ] API密钥管理安全（环境变量配置）
- [ ] 跨域配置正确（CORS）
- [ ] 静态资源路径正确
- [ ] 文件上传下载功能正常

## 配置集成检查
- [ ] 环境配置文件完整（开发、测试、生产）
- [ ] 数据库连接配置正确
- [ ] 日志配置一致
- [ ] 缓存配置一致
- [ ] 消息队列配置一致（如使用）

## 安全集成检查
- [ ] 身份认证机制一致（JWT token格式）
- [ ] 权限验证机制一致
- [ ] 输入验证机制一致
- [ ] 输出过滤机制一致
- [ ] SQL注入防护启用
- [ ] XSS防护启用

## 部署集成检查
- [ ] 依赖包版本一致
- [ ] 构建脚本正确
- [ ] 容器配置正确（Dockerfile）
- [ ] 环境变量配置正确
- [ ] 健康检查端点正常

## 检查频率
- **每日**: 接口一致性检查、数据格式检查
- **每周**: 安全集成检查、配置集成检查
- **每次部署前**: 全量集成检查
