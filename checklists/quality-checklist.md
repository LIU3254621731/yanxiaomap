# 质量检查清单

## 概述
- **用途**: 确保代码质量、设计质量、文档质量符合项目标准
- **适用阶段**: 整个项目生命周期
- **负责人**: 各Agent（自检），integration Agent（抽检）

## 代码质量检查
### 代码规范
- [ ] 代码格式统一（使用prettier/eslint）
- [ ] 命名规范一致（变量、函数、类名）
- [ ] 注释完整（函数、复杂逻辑）
- [ ] 文件组织合理（目录结构清晰）
- [ ] 导入/导出规范

### 代码复杂度
- [ ] 函数长度不超过50行
- [ ] 圈复杂度不超过10
- [ ] 重复代码不超过5%
- [ ] 嵌套深度不超过3层

### 错误处理
- [ ] 异常处理完整（try-catch）
- [ ] 错误信息友好
- [ ] 日志记录适当
- [ ] 资源释放正确（关闭连接、流）

## 设计质量检查
### 数据库设计
- [ ] 表结构符合第三范式
- [ ] 索引设计合理（查询性能）
- [ ] 外键约束正确
- [ ] 数据类型合适
- [ ] 默认值设置合理

### 架构设计
- [ ] 分层清晰（Controller/Service/DAO）
- [ ] 模块职责单一
- [ ] 依赖关系合理
- [ ] 扩展性考虑

### API设计
- [ ] RESTful规范遵循
- [ ] 资源命名合理
- [ ] 版本管理考虑
- [ ] 兼容性考虑

## 文档质量检查
### 技术文档
- [ ] 数据库设计文档完整
- [ ] API接口文档完整（Swagger/OpenAPI）
- [ ] 部署文档完整
- [ ] 架构设计文档完整

### 用户文档
- [ ] 用户手册完整
- [ ] 管理员手册完整
- [ ] 常见问题解答

## 测试质量检查
### 单元测试
- [ ] 测试覆盖率 > 80%
- [ ] 边界条件测试
- [ ] 异常场景测试
- [ ] Mock使用合理

### 集成测试
- [ ] 接口测试完整
- [ ] 数据流测试完整
- [ ] 跨模块测试完整

### 性能测试
- [ ] 响应时间达标
- [ ] 并发处理能力达标
- [ ] 资源使用合理

## 安全质量检查
- [ ] 身份认证安全
- [ ] 权限控制安全
- [ ] 数据加密安全
- [ ] 输入验证安全
- [ ] 输出过滤安全

## 数据质量检查
- [ ] 数据完整性
- [ ] 数据一致性
- [ ] 数据准确性
- [ ] 数据时效性

## 检查频率
- **提交前**: 代码质量检查（本地）
- **每日**: 设计质量检查（peer review）
- **每周**: 文档质量检查
- **里程碑**: 全面质量检查
