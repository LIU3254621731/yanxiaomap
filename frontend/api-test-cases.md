# 前端API接口测试用例（联调准备）

## 概述
本文档定义了前端与后端API接口的测试用例，用于前后端联调测试。每个接口包含正常场景和异常场景的测试用例。

## 测试环境
- 前端地址：http://localhost:5173
- 后端地址：http://localhost:8080
- API前缀：/api

## 接口测试用例

### 1. 高德地图配置接口
**接口路径**: GET /api/map/config

#### 测试用例1.1：正常获取配置
- **输入**: 无参数
- **预期响应**: 
  ```json
  {
    "success": true,
    "message": "成功",
    "data": {
      "apiKey": "f2a14d5a5748760eea937b4a756d6e81",
      "securityKey": "986e6e43d1e301521ba536d63153d51a",
      "version": "2.0",
      "plugin": [],
      "appName": "yanxiaomap"
    }
  }
  ```
- **验证点**:
  1. 响应状态码200
  2. success字段为true
  3. data字段包含apiKey和securityKey
  4. 地图能正常加载

#### 测试用例1.2：后端服务不可用
- **模拟场景**: 后端返回502错误
- **预期行为**: 前端使用默认配置回退，地图仍能加载
- **验证点**: 控制台应有错误日志，但地图功能正常

### 2. 院校地图数据接口
**接口路径**: GET /api/map/schools

#### 测试用例2.1：正常获取地图数据
- **输入**: 无参数（或可选参数bounds）
- **预期响应**: 
  ```json
  {
    "success": true,
    "message": "成功",
    "data": [
      {
        "id": 1,
        "name": "清华大学",
        "code": "10001",
        "level": "985",
        "province": "北京",
        "city": "北京市",
        "type": "综合类",
        "belong": "教育部",
        "longitude": 116.328,
        "latitude": 39.999,
        "website": "https://www.tsinghua.edu.cn",
        "logo": "/logos/tsinghua.png",
        "status": 1,
        "createdAt": "2024-01-01T00:00:00Z",
        "updatedAt": "2024-01-01T00:00:00Z"
      }
    ]
  }
  ```
- **验证点**:
  1. 响应状态码200
  2. data字段为School数组
  3. 地图上显示正确的标记数量
  4. 标记点击显示正确信息

#### 测试用例2.2：带边界筛选
- **输入**: bounds参数（地图当前视野边界）
- **预期响应**: 只返回边界内的院校
- **验证点**: 地图标记数量与返回数据一致

#### 测试用例2.3：空数据
- **模拟场景**: 无院校数据或边界内无院校
- **预期响应**: data字段为空数组
- **验证点**: 地图不显示任何标记

### 3. 院校详情接口
**接口路径**: GET /api/schools/{schoolId}

#### 测试用例3.1：正常获取院校详情
- **输入**: schoolId=1
- **预期响应**: 包含完整School对象，包含address和introduction字段
- **验证点**:
  1. 响应包含院校基本信息
  2. 页面正确显示院校详情
  3. 地址和简介字段显示正常

#### 测试用例3.2：院校不存在
- **输入**: schoolId=99999（不存在）
- **预期响应**: success=false, message="院校不存在"
- **验证点**: 页面显示错误提示，用户可返回

### 4. 院校-专业详情接口
**接口路径**: GET /api/schools/{schoolId}/majors/{majorId}

#### 测试用例4.1：正常获取专业详情
- **输入**: schoolId=1, majorId=1
- **预期响应**: SchoolMajorDetailResponse结构
- **验证点**:
  1. 响应包含school、major、schoolMajor、admissionHistory
  2. 页面正确显示专业信息
  3. 历年录取数据图表显示正常

#### 测试用例4.2：专业不存在
- **输入**: majorId=99999
- **预期响应**: success=false, message="专业不存在"
- **验证点**: 显示错误信息

#### 测试用例4.3：无录取数据
- **模拟场景**: admissionHistory为空数组
- **预期行为**: 页面显示"暂无录取数据"
- **验证点**: 图表区域显示空状态

### 5. 批量对比接口
**接口路径**: GET /api/compare/batch

#### 测试用例5.1：正常批量对比
- **输入**: requests=[{schoolId:1,majorId:1},{schoolId:2,majorId:3}]
- **预期响应**: SchoolMajorDetailResponse数组
- **验证点**:
  1. 返回数组长度与请求数量一致
  2. 每个元素包含完整的详情数据
  3. 对比页面正确显示所有院校专业

#### 测试用例5.2：空请求数组
- **输入**: requests=[]
- **预期响应**: data为空数组
- **验证点**: 对比页面显示空状态

#### 测试用例5.3：部分院校不存在
- **输入**: 包含一个不存在的schoolId
- **预期响应**: 成功返回存在的院校，不存在的返回null或错误
- **验证点**: 页面正确显示可用的对比数据

### 6. 连接状态测试接口
**接口路径**: GET /api/health

#### 测试用例6.1：服务正常
- **预期响应**: {"status": "UP", "timestamp": "2024-01-01T00:00:00Z"}
- **验证点**: 连接状态显示正常

#### 测试用例6.2：服务异常
- **模拟场景**: 后端服务宕机
- **预期响应**: 网络错误或超时
- **验证点**: 连接状态显示异常，前端有降级处理

## 端到端测试流程

### 流程1：完整用户路径
1. 访问首页，地图加载正常
2. 点击地图标记，打开信息窗口
3. 点击"查看详情"，进入院校详情页
4. 选择专业，查看专业详情
5. 点击"加入对比"，添加多个院校专业
6. 进入对比页面，查看对比数据
7. 返回首页，进行筛选操作

### 流程2：API异常处理路径
1. 模拟后端服务不可用
2. 验证前端降级机制（模拟数据回退）
3. 恢复服务，验证自动重连
4. 测试网络超时处理
