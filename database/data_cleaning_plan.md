## 概述

本方案针对考研院校地图择校网站的Excel数据源，制定数据清洗、转换和导入规则。数据源包含多个Excel文件，涵盖2017-2024年的考研招生目录、复试分数线、院校信息等数据。

## 数据库表与Excel字段映射

### 1. schools表 (院校信息)

**数据来源**：多个Excel文件中的院校相关字段

**字段映射**：
- `name` ← `院校名称`、`学校`、`学校名称` (需去重和标准化)
- `code` ← `院校代码` (从招生目录文件中提取)
- `province` ← `所在地`、`学校省份`、`所在城市` (需解析省份信息)
- `city` ← 从`学校地址`或`所在城市`中提取
- `level` ← `院校层级`、`学校属性`、`985`、`211`、`双一流` (需解析为标签)
- `type` ← `院校类型`、`学校属性`中的类型信息
- `is_985` ← `985`字段 (是/否 → 1/0)
- `is_211` ← `211`字段 (是/否 → 1/0)
- `is_double_first_class` ← `双一流`字段 (是/否 → 1/0)
- `is_self_line` ← `自划线`字段 (是/否 → 1/0)
- `zone` ← `AB区`字段 (A区/B区)
- `official_website` ← `招生官网`、`学校官网`
- `admission_website` ← `学校研究生官网`
- `phone` ← `招生电话`、`学校电话`
- `email` ← `学校邮箱`
- `address` ← `学校地址`
- `affiliation` ← `隶属`
- `master_programs_count` ← `硕士点`
- `doctoral_programs_count` ← `博士点`
- `national_key_disciplines` ← `国家重点学科`
- `national_key_laboratories` ← `国家重点实验室`、`重点实验室`
- `longitude`、`latitude` ← 需要从地址中解析或通过地理编码API获取

**清洗规则**：
1. 院校名称标准化：去除括号内容，如"(10001)北京大学" → "北京大学"
2. 省份提取：从`所在地`字段中提取省份，如"(11)北京市" → "北京"
3. 城市提取：从`学校地址`中提取城市名
4. 标签解析：`院校层级`字段如"985,211,双一流"需解析为单独的布尔字段
5. 数字字段处理：空值转为0
6. 网址规范化：确保以http://或https://开头

### 2. subject_categories表 (学科门类)

**数据来源**：招生目录文件中的`门类代码`、`门类名称`

**字段映射**：
- `code` ← `门类代码`
- `name` ← `门类名称`

**清洗规则**：
1. 去重：相同代码的学科门类只保留一条
2. 名称标准化：去除多余空格和特殊字符

### 3. disciplines表 (一级学科)

**数据来源**：招生目录文件中的`学科代码`、`学科领域`

**字段映射**：
- `code` ← `学科代码`
- `name` ← `学科领域`
- `subject_category_id` ← 外键，关联`门类代码`

**清洗规则**：
1. 去重：相同代码的学科只保留一条
2. 关联学科门类：根据同一行数据中的`门类代码`建立关联

### 4. majors表 (专业)

**数据来源**：多个文件中的`专业代码`、`专业名称`

**字段映射**：
- `code` ← `专业代码`
- `name` ← `专业名称`
- `discipline_id` ← 外键，关联`学科代码`
- `degree_type` ← `学位性质`、`专硕/学硕`、`硕士类型` (学硕/专硕)
- `exam_method` ← `考试方式` (统考/自主命题等)

**清洗规则**：
1. 专业代码清洗：去除括号和多余字符，如"(010101)马克思主义哲学" → "010101"
2. 专业名称清洗：提取括号内的标准名称，如"(010101)马克思主义哲学" → "马克思主义哲学"
3. 学位类型标准化：`学位性质`、`专硕/学硕`、`硕士类型`统一为"学硕"或"专硕"
4. 考试方式标准化：统一为"统考"、"自主命题"等标准值

### 5. school_majors表 (院校-专业关联)

**数据来源**：所有包含院校和专业信息的文件

**字段映射**：
- `school_id` ← 外键，关联`院校名称`
- `major_id` ← 外键，关联`专业代码`
- `department` ← `招生院系`、`院系所`、`所属院系`、`院系名称`
- `research_direction` ← `研究方向`
- `study_mode` ← `学习方式` (全日制/非全日制)
- `advisor` ← `指导教师`、`指导老师`
- `exam_subject1` ← `业务课一`、`专业课一`
- `exam_subject2` ← `业务课二`、`专业课二`
- `foreign_language` ← `外语`、`英语`
- `political` ← `政治`、`政治综合`

**清洗规则**：
1. 院系名称清洗：去除括号和代码，如"(023)哲学系" → "哲学系"
2. 研究方向清洗：去除括号和代码，如"(01)马克思主义哲学史" → "马克思主义哲学史"
3. 学习方式标准化：统一为"全日制"、"非全日制"
4. 考试科目清洗：提取科目名称，如"(101)思想政治理论" → "思想政治理论"

### 6. admission_data表 (招生数据)

**数据来源**：多个文件中的招生和分数线数据

**字段映射**：
- `school_major_id` ← 外键，关联school_majors表
- `year` ← `年份`
- `plan_enroll` ← `拟招生人数`、`招生人数`、`拟招人数` (需解析数字)
- `actual_enroll` ← 实际招生人数 (部分文件可能缺失，需估算)
- `recommended_免试人数` ← 推免人数 (目前文件缺失，需后续补充或设为NULL)
- `application_to_admission_ratio` ← 报录比 (目前文件缺失，需后续补充或设为NULL)
- `retest_total_score` ← `总分` (复试总分线)
- `retest_politics_score` ← `政治`、`政治__管综` (政治单科线)
- `retest_foreign_language_score` ← `外语`、`英语` (外语单科线)
- `retest_professional1_score` ← `专业课一`、`业务课_一` (专业课一单科线)
- `retest_professional2_score` ← `专业课二`、`业务课_二` (专业课二单科线)
- `average_admission_score` ← 录取平均分 (目前文件缺失，需后续补充或设为NULL)
- `notes` ← `备注`、`招生人数说明`

**清洗规则**：
1. 招生人数解析：从字符串中提取数字，如"专业：2" → 2，"专业：4(不含推免)" → 4
2. 分数处理：空值或"-"转为NULL
3. 年份范围：确保年份在2017-2024之间
4. 数据完整性：缺失字段设为NULL，标记为需要后续补充

## 数据清洗步骤

### 第一阶段：基础数据清洗

1. **院校数据清洗**：
   - 从所有Excel文件中提取院校信息
   - 标准化院校名称、省份、城市
   - 解析标签字段(985/211/双一流等)
   - 去重合并，生成唯一的院校列表

2. **学科门类和一级学科清洗**：
   - 从招生目录文件提取学科门类和一级学科
   - 建立学科门类与一级学科的层级关系
   - 去重处理

3. **专业数据清洗**：
   - 从所有文件提取专业信息
   - 清洗专业代码和名称
   - 关联到对应的一级学科
   - 标准化学位类型和考试方式

### 第二阶段：关联数据清洗

4. **院校-专业关联清洗**：
   - 建立院校与专业的关联关系
   - 清洗院系、研究方向、学习方式等信息
   - 处理考试科目信息

5. **招生数据清洗**：
   - 提取历年招生计划和分数线数据
   - 解析招生人数字符串
   - 处理分数数据
   - 关联到对应的院校-专业组合

### 第三阶段：数据整合和验证

6. **数据完整性检查**：
   - 检查外键关联完整性
   - 验证数据范围和逻辑一致性
   - 标记缺失数据字段

7. **数据去重和合并**：
   - 合并不同来源的相同数据
   - 处理冲突数据（以最新或最完整数据为准）
   - 生成最终导入数据集

## 风险与应对

### 风险1：数据缺失
- **表现**：部分字段在Excel中缺失（如推免人数、报录比）
- **应对**：标记为NULL，后续通过其他渠道补充或使用默认值

### 风险2：数据不一致
- **表现**：同一院校或专业在不同文件中信息不一致
- **应对**：制定优先级规则（以最新、最完整数据为准），记录冲突解决日志

### 风险3：数据量过大
- **表现**：Excel文件数量多，数据量大
- **应对**：分批处理，使用内存优化技术，增加进度监控

### 风险4：清洗规则不完善
- **表现**：某些特殊情况未覆盖
- **应对**：增加异常处理，记录清洗日志，支持规则迭代优化
