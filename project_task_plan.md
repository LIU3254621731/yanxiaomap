# 考研院校地图择校网站 - 任务规划

## 项目概述
基于需求文档《研校地图-开发文档.md》，本项目是一个地图交互式考研择校平台，整合全国研究生院校2017-2024年招生录取数据，提供地图交互、多维度筛选、院校详情、多校对比等核心功能。

## 技术栈
- **前端**: Vue3 + Element Plus + 高德地图JS API + Axios + Vite
- **后端**: Spring Boot + MyBatis-Plus + MySQL 8.0 + Redis + Spring Security + JWT
- **数据库**: MySQL 8.0（9张核心表）
- **部署**: Docker + 阿里云/腾讯云轻量应用服务器

## 任务拆解原则
1. **并行开发**: 前端、后端、数据库可同时开工，通过接口定义和共享状态文件协调
2. **接口驱动**: 前后端分离，先定义API接口规范，再各自实现
3. **数据先行**: 数据库设计先行，提供数据模型供前后端使用
4. **安全内置**: 安全合规需求贯穿所有开发任务
5. **渐进增强**: 先完成MVP核心功能，再考虑扩展功能

## 接口定义协调
为确保前后端并行开发顺利，需要优先定义以下核心接口：

### 1. 地图数据接口
```
GET /api/map/schools
参数: province? (省份筛选), level? (院校层次), keyword? (专业关键词)
响应: { schools: [{ id, name, longitude, latitude, level, province, city, majors: [{ name, year, score }] }] }
```

### 2. 筛选查询接口
```
GET /api/schools/search
参数: keyword? (专业名), province? (省份), level? (院校层次), sortBy? (score/enroll/ratio), order? (asc/desc), page?, size?
响应: { total, schools: [{ id, name, level, province, majorName, year, score, enroll, ratio }] }
```

### 3. 院校专业详情接口
```
GET /api/schools/{schoolId}/majors/{majorId}
响应: { school: {基本信息}, major: {专业信息}, admissionData: [{ year, planEnroll, actualEnroll, score, ratio }] }
```

### 4. 多校对比接口
```
POST /api/compare
参数: { schoolIds: [id1, id2, id3, id4], majorId: 专业ID }
响应: { schools: [对比数据], comparison: {差异分析} }
```

## 开发约束与限制

### 技术约束
1. 必须使用文档中指定的技术栈，不得随意替换
2. 前端必须兼容Chrome/Firefox/Edge最新版
3. 后端API响应时间必须<500ms（95分位）
4. 数据库查询必须使用索引，避免全表扫描
5. 所有用户输入必须验证和转义

### 安全限制
1. 管理端接口必须认证授权
2. 敏感数据不得明文存储
3. 接口必须有速率限制
4. 错误信息不得泄露系统细节
5. 必须部署HTTPS

### 合规要求
1. 网站底部必须展示备案信息
2. 数据来源必须明确声明
3. 用户隐私政策必须明确
4. 必须遵守国内互联网监管要求

## 风险与应对

### 技术风险
1. **高德地图API限制**: 免费版有调用次数限制
   - 应对: 前端缓存地图数据，后端聚合请求

2. **数据量较大**: 2017-2024年全国院校专业数据
   - 应对: 数据库分页查询，Redis缓存热门数据

3. **前后端接口不一致**: 并行开发可能导致接口不一致
   - 应对: 先定义接口规范，使用Mock数据开发，集成协调器定期检查

### 管理风险
1. **进度不同步**: 某个任务延误影响整体
   - 应对: 每日站会（通过共享状态文件），集成协调器及时协调

2. **需求理解偏差**: 不同Agent对需求理解不同
   - 应对: 需求文档作为唯一标准，关键决策点记录在共享文件中
